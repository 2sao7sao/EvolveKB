# Security Policy

EvolveKB is designed for governed knowledge workflows, but the repository is a
prototype and should not be treated as a hardened enterprise system.

## Do Not Commit

- API keys, tokens, cookies, or credentials.
- Raw customer documents or private incident traces.
- Proprietary policy documents that have not been sanitized.
- Generated proposals that contain private data.

## Reporting

Open a GitHub issue for non-sensitive security hardening suggestions. For
sensitive vulnerabilities, contact the repository owner privately instead of
posting exploit details publicly.

## Current Boundaries

- No production permission model is included.
- No enterprise document ACL enforcement is included.
- Demo data is synthetic and should remain synthetic.
